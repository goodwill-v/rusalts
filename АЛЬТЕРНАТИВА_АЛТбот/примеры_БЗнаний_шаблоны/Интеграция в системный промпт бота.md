# 🎯 РАБОТА С ШАБЛОНАМИ

## Источник шаблонов
- Шаблоны хранятся в: `templates/invitations_ru.json`
- Триггеры для вызова: `knowledge_base/triggers.json`
- НЕ ищи текст шаблонов в Базе знаний — они вынесены в отдельный конфиг

## Алгоритм ответа
1. Получи запрос пользователя
2. Найди matching trigger в `triggers.json` по ключевым словам
3. Загрузи шаблон из `templates/invitations_ru.json` по `template_key`
4. Подставь переменные из раздела `variables` (app_link, channel_link, promo_code и т.д.)
5. Отправь пользователю

## Переменные для подстановки
| Переменная | Значение |
|-----------|----------|
| `{app_link}` | https://max.ru |
| `{channel_link}` | https://vk.com/im/medialogic/АЛЬТЕРНАТИВА |

| `{promo_code}` | MAX2026 |
| `{bonus}` | чек-лист + персональный план миграции |
| `{expiry_date}` | 30.06.2026 |

## Важные правила
- ✅ Если trigger найден — используй шаблон ТОЧНО как в конфиге (можно менять только переменные)
- ✅ Если trigger не найден — используй fallback из `triggers.json`
- ✅ Логируй использование: trigger_id, template_key, timestamp
- ❌ НЕ выдумывай новые шаблоны на лету
- ❌ НЕ меняй текст шаблонов без обновления конфигурационного файла
