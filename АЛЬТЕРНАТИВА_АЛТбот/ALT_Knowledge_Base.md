# 📘 База знаний ассистента МАХ (Актуализация: Март 2026)

## 1. О платформе МАХ
**МАХ** — многофункциональный сервис обмена информацией, разработанный в рамках стратегии цифрового суверенитета РФ.
- **Основная цель:** Замена зарубежных мессенджеров для госструктур, бизнеса и граждан.
- **Ключевая особенность:** Глубокая интеграция с экосистемой Госуслуг (ЕСИА) и отечественным ПО.
- **Статус:** Приоритетное ПО для предустановки на устройства в РФ.
- **Аудитория:** 85+ млн зарегистрированных пользователей, 55+ млн ежедневных активных пользователей (данные на начало 2026).

---

## 2. Переход для пользователей (Миграция)

### Как начать работу
1. **Скачивание:** Только из отечественных сторов (RuStore, NashStore) или с официального сайта.
2. **Регистрация:** По номеру телефона РФ (+7). Виртуальные номера не поддерживаются.
3. **Верификация:** Рекомендуется подтверждение через Госуслуги для получения статуса «Проверенный пользователь» (галочка).

### Перенос данных
- **Контакты:** Автоматическая синхронизация из телефонной книги (с согласия пользователя).
- **История переписки:** Прямой перенос из WhatsApp/Telegram технически невозможен из-за различий в протоколах шифрования. Рекомендуется использование функции «Экспорт чата» в файлах на устройстве.

## 3. 🔄 Миграция из других мессенджеров: стратегия сохранения аудитории

> ⚠️ **Важно:** Прямого «переезда одной кнопкой» не существует. Подписчики, контент и боты требуют отдельной работы. Ниже — проверенные сценарии.

### 🔹 Что можно перенести технически

| Элемент | Способ переноса | Ограничения |
|---------|----------------|-------------|
| **Посты (текст, фото, видео)** | Кросспостинг-боты или экспорт архива | Форматирование может требовать ручной правки |
| **Архив переписки / группы** | Экспорт ZIP из Telegram Desktop → отправка файлом в МАХ | Только для чтения, не встраивается в интерфейс чата |
| **Чат-бот (логика воронки)** | Ручное воссоздание или конвертация через конструкторы | Платформенно-специфичные элементы (inline-кнопки TG) требуют адаптации |
| **Подписчики** | Только добровольный переход по приглашению | Нельзя «перенести» базу — каждый пользователь должен сам подписаться |
| **Аналитика** | Ручной экспорт / настройка аналитики в МАХ с нуля | Автоматического импорта метрик нет |

### 🔹 Пошаговая стратегия перехода для бизнеса

#### Шаг 1: Подготовка контента
- **Кросспостинг:** Настройте бота-дубликатора для автоматического копирования новых постов из Telegram в МАХ (на переходный период).
- **Тестовый прогон:** Перед массовым переносом протестируйте 10–20 постов — проверьте отображение форматирования, эмодзи, вложений.
- **Архив:** Экспортируйте историю канала через Telegram Desktop (⋯ → «Экспортировать историю чата») и сохраните как резервную копию.

#### Шаг 2: Создание канала в МАХ
1. Зарегистрируйтесь в [business.max.ru](https://business.max.ru) как юрлицо/ИП РФ.
2. Пройдите верификацию через Госуслуги / Альфа-ID / Т-Business ID (автоматически — несколько минут, вручную — до 4 дней).
3. Создайте канал организации: загрузите аватар, описание, контакты.
4. Настройте приветственное сообщение и закрепите пост с инструкцией для новых подписчиков.

#### Шаг 3: Приглашение аудитории
- **Ссылка / QR-код:** Опубликуйте в старом канале призыв с прямой ссылкой на канал в МАХ.
- **Ценностный оффер:** Дайте конкретный повод перейти:
  - Эксклюзивный контент / ранний доступ к новинкам
  - Бонус за подписку (чек-лист, скидка, доступ к закрытому чату)
  - Удобство: «Все сервисы в одном приложении с Госуслугами»
- **Мягкий переход:** Не удаляйте старый канал сразу — ведите параллельно 2–4 недели, постепенно смещая акцент на МАХ.

#### Шаг 4: Адаптация чат-бота
- **Логика:** Перенесите сценарии диалогов вручную или через конструкторы с поддержкой МАХ.
- **Элементы:** Замените Telegram-специфичные функции (inline-режим, callback_data) на аналоги МАХ (кнопки, Web App).
- **Тестирование:** Обязательно протестируйте бота в песочнице перед запуском на реальную аудиторию.

#### Шаг 5: Мониторинг и обратная связь
- Отслеживайте активность новых подписчиков в МАХ.
- Собирайте обратную связь: что удобно, что вызывает сложности.
- Корректируйте контент-план под особенности платформы (форматы, длина сообщений, интерактивы).

### 🔹 Типичные ошибки при миграции
❌ **Ожидание «волшебной кнопки»** — тратьте время на стратегию, а не на поиск несуществующих инструментов.
❌ **Резкое отключение старого канала** — аудитория не успеет перейти, вы потеряете охваты.
❌ **Игнорирование форматирования** — посты могут «поехать», проверяйте отображение перед публикацией.
❌ **Отсутствие оффера** — без мотивации пользователи не будут устанавливать новое приложение.

### 🔹 Преимущества МАХ для удержания аудитории
✅ **Интеграция с Госуслугами** — доверие пользователей, упрощённая авторизация.
✅ **Оплата через СБП** — удобные транзакции внутри чата без перехода во внешние приложения.
✅ **Встроенный ИИ (GigaChat)** — готовые сценарии поддержки без подключения сторонних сервисов.
✅ **Мини-приложения** — полноценные веб-интерфейсы (каталоги, формы, сервисы) внутри мессенджера.
✅ **Государственная поддержка** — приоритетное развитие, льготы для резидентов РФ.

## 4. Для разработчиков: Барьеры входа (Март 2026)

### Организационные требования
- **Юридическое лицо:** Разработка коммерческих ботов и мини-приложений доступна только для резидентов РФ (ООО, ИП).
- **Идентификация:** Обязательная привязка проекта к цифровому профилю компании на Госуслугах.
- **Маркировка:** Каналы и боты с аудиторией >10 000 подписчиков подлежат регистрации в реестре РКН и маркировке (А+).

### Технические барьеры
- **Сертификация:** Приложения для госсектора должны иметь сертификат ФСТЭК (для работы с персональными данными 1-го уровня).
- **Хранение данных:** Все данные пользователей (включая метаданные) должны храниться на серверах внутри РФ (требование 152-ФЗ).
- **Шифрование:** Использование отечественных криптографических алгоритмов (ГОСТ) для защищенных каналов связи.
- **Совместимость:** Требуется поддержка отечественных ОС (Astra Linux, Alt Linux, Aurora OS) для корпоративных клиентов.

### Финансовые вопросы
- **Комиссия:** Для резидентов РФ комиссия за платежи через ботов снижена до 0% (льготный период до 2027 года).
- **Прием платежей:** Только через СБП (Система быстрых платежей) или российские платежные шлюзы.

## 7. Частые вопросы (FAQ)

**В: Можно ли полностью перенести канал из Telegram в МАХ?**  
О: Нет прямого импорта. Посты копируются через кросспостинг-ботов или вручную, но подписчики переходят только добровольно — по приглашению. По аудитории придётся начинать с нуля.

**В: Как перенести подписчиков из Telegram в МАХ-бот?**  
О: Технически это невозможно — подписчик должен сам написать боту в МАХ или подписаться на рассылку. Разместите ссылку на МАХ-бота в Telegram-канале и дайте людям повод перейти: эксклюзивный контент, удобство, акция.

**В: Сколько времени занимает перенос постов через кросспостинг?**  
О: Канал до 500 постов переносится за 2–15 минут. Скорость зависит от объёма медиафайлов и нагрузки на серверы.

**В: Можно ли вести Telegram и МАХ параллельно?**  
О: Да, и это рекомендуемая стратегия на переходный период. Настройте автоматическое дублирование: новые посты в Telegram будут появляться и в МАХ.

**В: Что происходит при адаптации бота из Telegram в МАХ?**  
О: Переносятся логика, тексты, кнопки, переходы. Платформенно-специфичные функции, которых нет в МАХ, удаляются — их нужно настроить заново под доступные элементы API.

**В: Нужно ли отдельно регистрировать бизнес-аккаунт в МАХ, если есть аккаунт в другом мессенджере?**  
О: Да. МАХ требует отдельную регистрацию для юрлиц и ИП РФ с верификацией через Госуслуги или банковские сервисы.

**В: Поддерживает ли МАХ форматирование постов из Telegram?**  
О: Базовое форматирование текста, эмодзи, изображения и видео переносятся корректно. Telegram-специфичные элементы вроде inline-кнопок могут отображаться иначе или потребовать дополнительной настройки.

**В: Можно ли использовать МАХ за границей?**  
О: Да, функционал доступен, но некоторые сервисы (платежи, Госуслуги) работают только для резидентов РФ.

**В: Есть ли сквозное шифрование (E2EE)?**  
О: Для личных чатов — опционально. Для рабочих чатов и ботов — сообщения обрабатываются на сервере для обеспечения безопасности и модерации.

**В: Сколько стоит размещение бота?**  
О: Размещение базового бота бесплатно. Плата взимается только с транзакций внутри платных мини-приложений.

## 🔐 3. Цифровой ID: подтверждение возраста и льготного статуса

### Что это даёт бизнесу
| Возможность | Бизнес-результат |
|-------------|-----------------|
| Быстрая верификация возраста (18+, 21+) | Соответствие требованиям при продаже алкоголя, табака, игр |
| Подтверждение льготного статуса (пенсионер, студент, многодетный) | Автоматическое применение скидок без запроса документов |
| Юридически значимая идентификация через Госуслуги | Снижение риска мошенничества и возвратов |
| Интеграция с «Госключом» для ЭДО | Подписание договоров и актов прямо в чате |

### Как это работает
1. Пользователь нажимает кнопку «Подтвердить через Цифровой ID» в боте или мини-приложении
2. Открывается окно авторизации через Госуслуги / Тинькофф / Альфа-Банк
3. После успешной аутентификации бот получает подтверждённый атрибут (например, `age_verified: true`)
4. Бизнес-логика адаптируется под статус пользователя (показывает/скрывает товары, применяет скидку)
