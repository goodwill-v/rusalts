# Подключение мини-приложения (виджета) ВКонтакте к веб-проекту

Инструкция основана на проекте **АЛЬТЕРНАТИВА**: сайт `https://rusalts.ru/`, страница виджета `https://rusalts.ru/widget`, бэкенд FastAPI за Nginx. Логика переносится на любой стек при сохранении **HTTPS**, **инициализации VK Bridge** и корректных **заголовков безопасности**.

---

## 1. Что должно быть до кабинета VK

1. **Публичный домен с HTTPS**  
   ВК открывает приложение по URL из браузера / WebView. Адрес вида `http://localhost` в настройках приложения обычно не подходит для продакшена; для отладки с ПК часто используют туннель (ngrok и т.п.) с HTTPS.

2. **Рабочая страница виджета по фиксированному пути**  
   В текущем проекте это **`https://rusalts.ru/widget`**. У вас может быть, например, **`https://<ваш-домен>/<путь-к-виджету>`**.

3. **Сервер отдаёт страницу без запрета встраивания во фрейм**  
   Не используйте `X-Frame-Options: DENY` для этой страницы. Для Content-Security-Policy задайте **`frame-ancestors`**, разрешающий домены ВКонтакте и при необходимости свой сайт (см. раздел про переменные окружения).

4. **Инициализация мини-приложения в коде**  
   Обязателен вызов **`VKWebAppInit`** через официальный пакет **`@vkontakte/vk-bridge`**. Без этого клиент VK часто показывает сообщение вроде «Приложение не инициализировано».

---

## 2. Регистрация приложения в VK

1. Войдите под аккаунтом разработчика: **[Мои приложения](https://vk.com/apps?act=manage)**.

2. Создайте приложение подходящего типа (мини-приложение / веб-приложение — ориентируйтесь на актуальные варианты в интерфейсе VK).

3. Запишите **ID приложения** (число). В коде и в `.env` он соответствует переменной **`VK_APP_ID`**.  
   Пример для текущего проекта: приложение **«Консультант АЛТ»**, ID **54525512** (у вас будет свой).

4. Откройте раздел **«Ключи доступа»**:
   - **Защищённый ключ** → в проекте хранится как **`VK_SECURE_KEY`** (только на сервере, не в браузере и не в репозитории).
   - **Сервисный ключ** → **`VK_SERVICE_TOKEN`** (только на сервере).

5. Статус **«Непроверенное приложение»** для первых тестов в своём сообществе часто допустим: ограничение «доступно по ссылке» в основном про способ распространения, а не про полный запрет тестирования. Требования могут меняться — ориентируйтесь на подсказки в кабинете VK.

---

## 3. Настройка «Размещение» (URL для iframe и WebView)

В настройках приложения найдите блок вроде **«Размещение»** / **«Приложения загружаются в iframe»** и **«Мобильное приложение»** (WebView).

Заполните **один и тот же HTTPS-URL** виджета:

| Место в интерфейсе VK | Что указать (пример текущего проекта) |
|------------------------|--------------------------------------|
| Десктопная версия сайта (iframe) | `https://rusalts.ru/widget` |
| Мобильная версия сайта (iframe) | `https://rusalts.ru/widget` |
| Мобильное приложение (WebView) | `https://rusalts.ru/widget` |

Универсальный шаблон: **`https://<ваш-домен>/<путь-виджета>`**.

Пустой URL в секции **WebView** приводит к тому, что часть клиентов VK не откроет приложение так же, как десктоп.

## 5. Серверное приложение: что важно в коде

### 5.1. VK Bridge и `VKWebAppInit`

На странице виджета нужно подключить **`@vkontakte/vk-bridge`** (например, с CDN **`unpkg.com`**) и после загрузки вызвать:

```text
vkBridge.send("VKWebAppInit")
```

При необходимости затем вызывают **`VKWebAppResize`** и другие методы [документации VK](https://dev.vk.com/ru).

### 5.2. Content-Security-Policy (CSP)

Если задаёте CSP глобально или для страницы виджета:

- **`frame-ancestors`** — должен включать домены VK (`vk.com`, `*.vk.com`, `vk.ru`, …) и при встраивании на своём сайте — **`'self'`** и ваш домен (например `https://rusalts.ru`).
- **`script-src`** — разрешить источник скрипта vk-bridge (например `https://unpkg.com` и/или `https://*.unpkg.com`), плюс домены VK для их скриптов.
- **`connect-src`** — для работы моста и запросов внутри контейнера VK часто нужны `https://*.vk.com`, `oauth.vk.com`, `api.vk.com`, при необходимости **wss:**.

Точный список лучше уточнять по консоли браузера при ошибках CSP.

### 5.3. Прокси (Nginx и т.д.)

Бэкенд (например uvicorn на `127.0.0.1:8000`) должен быть доступен снаружи по HTTPS на путях **`/widget`**, **`/api`**, **`/static`** — как в вашем конфиге **`deploy/nginx-rusalts.ru.conf`**. Шаблон пути: **`https://<домен>/widget`**.

## 6. Переменные окружения (шаблон)

Скопируйте в **`.env`** на сервере и подставьте свои значения. Примеры значений текущего проекта даны для наглядности; для нового проекта используйте **плэйсхолдеры**.

```env
# Публичный адрес сайта (без слэша в конце), тот же домен, что в браузере.
PUBLIC_BASE_URL=https://rusalts.ru

# ID приложения из кабинета VK (число).
VK_APP_ID=<ID_ПРИЛОЖЕНИЯ_VK>

# Защищённый ключ — см. раздел 7.
VK_SECURE_KEY=<ЗАЩИЩЁННЫЙ_КЛЮЧ_ИЗ_VK>

# Сервисный ключ — см. раздел 7.
VK_SERVICE_TOKEN=<СЕРВИСНЫЙ_КЛЮЧ_ИЗ_VK>

# CORS: ваш домен + домены VK (через запятую, без пробелов или с пробелами — как принято в вашем коде).
CORS_ORIGINS=https://rusalts.ru,https://www.rusalts.ru,https://vk.com,https://vk.ru,https://m.vk.com

# CSP frame-ancestors: в двойных кавычках целиком значение — удобно для python-dotenv.
FRAME_ANCESTORS="'self' https://rusalts.ru https://www.rusalts.ru https://vk.com https://*.vk.com https://vk.ru https://*.vk.ru https://m.vk.com"
```

Для **другого домена** замените `rusalts.ru` на **`<ваш-домен>`**.

После изменения `.env` перезапустите процесс приложения (например **`systemctl restart alt-consultant`** на текущем сервере).

## 7. Назначение ключей и ID (перспектива)

| Поле в кабинете VK | Переменная в проекте | Нужно ли для «просто открыть виджет» | Зачем дальше |
|--------------------|----------------------|--------------------------------------|--------------|
| Числовой **ID приложения** | **`VK_APP_ID`** | Не обязательно для отрисовки iframe, если инициализация через vk-bridge уже есть | Привязка сценариев, отладка, часть SDK; удобно хранить в конфиге |
| **Защищённый ключ** | **`VK_SECURE_KEY`** | Нет | Проверка подписи параметров запуска (**`vk_*`**), защита от подделки запросов к вашему бэкенду, интеграции с Callback API |
| **Сервисный ключ** | **`VK_SERVICE_TOKEN`** | Нет | Вызовы **VK API с сервера** от имени приложения (строго на бэкенде, не в браузер) |

Итого: чтобы **страница виджета загрузилась** во ВКонтакте, критичны **HTTPS**, **URL в настройках VK**, **VKWebAppInit** и **корректные CSP/прокси**. Ключи **`VK_SECURE_KEY`** и **`VK_SERVICE_TOKEN`** подключают **безопасность и серверные интеграции с API VK**, а не «включение» iframe как такового.

## 8. Чек-лист при проблемах

1. Открывается ли **`https://rusalts.ru/widget`** (или ваш URL) в обычном браузере с **200** и без ошибок в консоли?
2. В кабинете VK заполнены **все три** URL (десктоп iframe, мобильный iframe, WebView)?
3. В консоли нет блокировок **CSP** по `script-src` / `connect-src`?
4. В логах бэкенда и Nginx нет **502** и обрывов прокси?
5. После смены ключей в VK обновите **`.env`** и перезапустите сервис.
