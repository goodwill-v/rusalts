## Релиз: очередь и воркеры — надёжный межсервисный конвейер

В АЛТ‑эксперт появился “экологичный” способ связывать агентов/сервисы: **очередь + воркеры**.

### Что сделали

- Внедрили очередь на базе Redis Streams для обмена задачами между сервисами.
- Развели сервисы на отдельные процессы:
  - **Parser worker** выполняет мониторинг и формирует change package;
  - **Content worker** забирает change package и готовит публикации.
- Добавили consumer groups и ack‑механику для повторной обработки при сбоях.

### Почему это важно для бизнеса

- **Надёжность**: задачи не теряются при рестартах, воркер можно перезапустить без потери очереди.
- **Масштабирование**: можно добавлять воркеров (Parser/Content) без изменения API.
- **Изоляция рисков**: сбой парсинга не “роняет” консультационный интерфейс и наоборот.
- **Готовность к продакшену**: проще вводить расписание (05:00 UTC) и контролировать SLA.

### Как это работает (коротко)

- Backend кладёт задачу “запусти парсер” в очередь.
- Parser worker делает проход по источникам и сохраняет `change_package.json`.
- Content worker получает событие и генерирует текст релиза/поста, публикует (в зависимости от режима согласования).
