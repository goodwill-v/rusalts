# Приложение АЛТ‑эксперт (прототип)

АЛТ‑эксперт — консультационное приложение АЛЬТЕРНАТИВА по вопросам использования российских мессенджеров (в первую очередь **MAX**): миграция с зарубежных платформ, шаблоны действий/документов, актуальные изменения требований и законодательства, новости и релизы.

Сейчас в репозитории уже есть прототип:
- веб‑страница (виджет) `GET /widget` для встраивания во ВКонтакте (iframe / мини‑приложение);
- API `POST /api/chat` (пока заглушка) и выдача файлов‑шаблонов `GET /api/document-templates`.

## Быстрый старт (локально)

Требования: Python 3.11+.

```bash
python -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
cp deploy/env.production.example .env
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Открыть:
- `http://127.0.0.1:8000/` — тестовая страница
- `http://127.0.0.1:8000/widget` — версия для VK
- `http://127.0.0.1:8000/docs` — OpenAPI

Проверка:

```bash
curl -sS http://127.0.0.1:8000/health
```

## Шаблоны документов (файлы)

Скачиваемые файлы лежат в каталоге `templates/document_templates/`.

Чтобы добавить шаблон:
- положите файл (например `.docx`, `.pdf`) в `templates/document_templates/`
- перезапустите сервер
- обновите страницу — файл появится в списке «Шаблоны документов»

## Настройка для ВКонтакте (виджет)

Краткая инструкция есть в `vidget_vk.md`.

Важное:
- **HTTPS** в продакшене
- страница **не должна** блокировать iframe (нужен корректный CSP `frame-ancestors`)
- на странице виджета вызывается `VKWebAppInit` через `vk-bridge` (уже сделано в `app/templates/widget.html`)

## Как работает main → heavy (роутинг моделей)

В проекте используется policy‑подход “**main → heavy**” для RouterAI:
- **main‑модель** — по умолчанию для большинства запросов (быстрее/дешевле).
- **heavy‑модель** — подключается только когда есть триггеры “высокий риск ошибки” или провал валидации результата.

Где настраивается:
- `.env.example` / `deploy/env.production.example`
  - Backend: `BACKEND_MODEL_MAIN`, `BACKEND_MODEL_HEAVY`
  - Content: `CONTENT_MODEL_MAIN`, `CONTENT_MODEL_HEAVY`

Триггеры эскалации (коротко):
- **Backend**: legal/комплаенс‑тематика в тексте запроса; низкая уверенность (мало KB‑хитов).
- **Content**: change package содержит `classification: legal.*`; либо ответ LLM не проходит валидацию (например, не удаётся распарсить ожидаемый JSON).

Логирование:
- в JSON‑логи пишутся `model`, `tokens_in/out`, `purpose` и `routing_reason` (почему выбрали main/heavy).
