# Техническое задание (сводное) — АЛТ‑эксперт

## 1. Общие требования

- Язык интерфейса: русский.
- Источники знаний: приоритет БЗ; при отсутствии — поиск по официальным ресурсам и пометка “требует уточнения”.
- Юридические вопросы: информационно, со ссылками на источники, без “юридической консультации”.
- Архитектура: 4 агента + тимлид, согласованные форматы данных.
- Экономия LLM: минимально достаточные модели, логирование токенов и стоимости.

## 2. Функции продукта (MVP)

### 2.1. Консультации
- прием вопросов через сайт/виджет VK;
- ответы с опорой на БЗ (и ссылки на источники);
- выдача чек‑листов и шаблонов действий по триггерам.

### 2.2. Шаблоны
- выдача текстовых шаблонов (JSON) и файлов‑шаблонов (docx/pdf);
- шаблоны не хранятся в БЗ, а живут в `templates/` (как в примерах).

### 2.3. Ежедневные обновления
- парсинг в 05:00 UTC;
- дифф, классификация изменений;
- обновление БЗ по секциям;
- формирование change package для Content.

### 2.4. Новости/релизы
- релизы по изменениям источников и по обновлениям продукта;
- публикация на сайт и в VK.

## 3. Нефункциональные требования

- Логи: JSON, `request_id`, `service`, `event`, LLM usage.
- Тесты: pytest + API контракты.
- Нагрузочное тестирование: k6 или locust, метрики P95/P99.
- Деплой: systemd + nginx reverse proxy; HTTPS обязательно.

## 4. Данные и форматы

- БЗ: атомарные статьи Markdown + YAML‑метаданные, секции по `section_path` (см. `docs/KNOWLEDGE_BASE.md`).
- Триггеры: JSON (`knowledge_base/triggers.json`).
- Шаблоны: JSON (`templates/*.json`) + переменные.
- Change package: JSON (Parser → Content).

## 5. Каналы коммуникации

MVP:

- VK mini app / виджет
