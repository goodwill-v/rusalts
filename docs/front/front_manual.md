# Руководство администратора фронтенда (rusalts.ru)

Документ описывает, **где и как вручную менять информацию на сайте**, какие технологии используются, и как устроены публичные/админ‑страницы.

---

## 1) Технологии и платформа

- **Backend/веб‑сервер**: FastAPI (Python)
  - Отдаёт HTML‑страницы через шаблоны Jinja2.
  - Отдаёт статику (CSS/JS/картинки) через `StaticFiles` по пути `/static/...`.
- **Frontend**: Jinja2‑шаблоны + “ванильные” CSS/JS (без React/Next).
- **Авторизация админ‑разделов**: HTTP Basic (логин/пароль) на уровне роутов FastAPI.

## 3) Где лежит фронтенд и что за что отвечает

### 3.1 Шаблоны страниц (HTML)
Папка: `app/templates/`

- `base.html`
  - Базовый каркас HTML (`<head>`, подключение `/static/css/app.css` и `/static/js/app.js`).
  - Имеет блоки:
    - `{% block body %}` — основное содержимое страницы
    - `{% block after_shell %}` — для **плавающих виджетов** вне `.shell` (важно для мобильного `position: fixed`)
- `index.html`
  - Главная страница.
  - Содержит:
    - Header + меню
    - Hero (шапка с фоновой картинкой)
    - основные блоки
    - правую колонку «Новости» (анонсы)
    - футер (иконки VK/MAX)
    - блок «АЛТ‑Чат» (окно чата) + круглая кнопка «Чат» — вынесено в `after_shell`
- `site_page.html`
  - Универсальный шаблон для “простых” страниц (`/news/`, `/laws/` и т.д.).
  - Отображает `page_h1` (заголовок сверху).
  - `page_description` выводится как HTML‑комментарий (техническая заметка, не публикуется на странице).
- `consultant.html`
  - Страница «АЛТ‑эксперт» (интерфейс чат + шаблоны).
- `widget.html`
  - VK‑версия интерфейса (инициализация `vkBridge`).

### 3.2 Статика (CSS/JS/изображения)
Папка: `app/static/`

- `app/static/css/app.css`
  - Все стили сайта, включая адаптив и стили “чата”.
- `app/static/js/app.js`
  - Логика UI:
    - открытие панели «АЛТ‑Чат» по кнопке «Чат» и закрытие по крестику
    - работа формы чата (отправка в `/api/chat`)
    - список шаблонов документов (для `/consultant`): загрузка из `/api/document-templates`
    - мобильное меню (бургер)
    - переключатель “для слабовидящих” (сейчас кнопка скрыта визуально, но функционал оставлен)
- `app/static/img/`
  - `logo.png` — логотип АЛТ
  - `cover.png` — фоновая шапка (фиолетовый баннер)

## 4) Как вручную менять контент на сайте

### 4.1 Меню и ссылки
- Главная страница: `app/templates/index.html`
- Универсальные страницы: `app/templates/site_page.html`

Чтобы изменить пункты меню:
- правьте блок `<nav id="site-nav" ...>` в нужном шаблоне.

### 4.2 Тексты на главной
Файл: `app/templates/index.html`

Типовые места:
- **Hero‑заголовок и подзаголовок** — внутри `<section class="hero">`.
- **Тексты карточек/аккордеонов/контактов** — соответствующие секции ниже.

### 4.3 Правая колонка «Новости» (анонсы на главной)
Файл: `app/templates/index.html`

Блок справа: `<aside id="news" ...>` → список `<ol class="news-list">`.

Формат одного элемента:
- дата: `<time datetime="YYYY-MM-DD">DD.MM.YYYY</time>`
- номер: `<span class="news-item__no">№ N</span>`
- анонс: `<p class="news-item__announce">...</p>`
- ссылка: `<a class="news-item__more" href="...">Показать ещё</a>`

Порядок отображения: **сверху — последние**.

### 4.4 Картинки (логотип и шапка)
Файлы:
- `app/static/img/logo.png`
- `app/static/img/cover.png`

Заменить можно простым перезаписыванием файлов (оставляя имена теми же), либо заменить пути в шаблонах:
- Главная: `app/templates/index.html`
- Универсальные страницы: `app/templates/site_page.html`

### 4.5 Кнопка «Чат» и окно «АЛТ‑Чат»
- Разметка: `app/templates/index.html` в блоке `{% block after_shell %}`.
- Стили: `app/static/css/app.css` (блоки `.altbot-root`, `.altbot`, `.altbot__fab`, `.altbot__panel`).
- Логика: `app/static/js/app.js` (функция `initAltExpertWidget()`).

Важно:
- Окно чата **не должно быть постоянно открытым** — оно скрыто атрибутом `hidden`.
- Открытие: по кнопке `data-open-altchat`.
- Закрытие: по кнопке с крестиком `data-close-chat`.

### 4.6 Страница «АЛТ‑эксперт» (`/consultant`)
Файл: `app/templates/consultant.html`

Что показывает:
- Чат (по API `/api/chat`)
- Список шаблонов документов (по API `/api/document-templates`)

Шаблоны документов:
- физические файлы лежат в `data/document_templates/`
- список и ссылки формирует бэкенд (`app/routers/api.py`, метод `GET /api/document-templates`)

## 5) Админ‑страницы (заготовки)

### 5.1 `/admin/`
Назначение: управление сайтом (будет расширяться).

### 5.2 `/publapprov/`
Назначение: модерация публикаций от роли “Контент”.

Планируемая логика (по ТЗ):
- список материалов в порядке поступления
- каждый материал в отдельном окне (редактируемый текст)
- действия:
  - **Одобрить** → публикация на `/news/` и анонс на главной + отправка финального текста “Контенту” для обучения/аналитики
  - **Закрепить** → “скрепка” и материал вверху списка
  - **Отмена** → убрать из очереди

На текущем этапе эти страницы — каркас (контент будет добавляться позднее).

## 6) Запуск проекта (локально)

Зависимости Python описаны в:
- `requirements.txt`

Типовой запуск:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Открыть:
- `http://127.0.0.1:8000/`

## 7) Переменные окружения (.env)

Пример: `.env.example`

Для фронтенда критично:
- `PUBLIC_BASE_URL` (используется в ссылках/интеграциях)
- `CORS_ORIGINS`, `FRAME_ANCESTORS` (для встраивания в iframe/VK)

Секреты (VK/RouterAI/SMTP и др.) **нельзя** размещать на фронтенде и в публичных файлах.
