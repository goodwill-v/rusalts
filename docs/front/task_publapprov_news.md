# Задание фронту: страницы `/publapprov/` и `/news/` (web‑одобрение без почты)

Цель: упростить процесс одобрения новостей — **без доставки писем Chief**. Материалы от `content-worker` попадают в **очередь** на `/publapprov/`, где их можно редактировать, одобрять, закреплять и отменять. Одобренные появляются на `/news/` и в анонсе на главной.

## Роли и доступ

- **`/publapprov/`**: техническая страница с Basic Auth (admin).
- **`/news/`**: общедоступная страница новостей.
- **Главная `/`**: справа блок «Новости» — анонсы.

## Источники данных (API)

### Очередь на согласование (только admin)

- **GET** `/api/content/queue`
  - Возвращает список `pending/needs_edit` в порядке: **закреплённые сверху**, далее по времени (новые выше).
  - Поля: `publication_id`, `created_at_utc`, `title`, `site_text`, `vk_text`, `pinned`, `status`, …

- **PUT** `/api/content/queue/{publication_id}`
  - Тело: `{ "title": "...", "site_text": "...", "vk_text": "..." }`

- **POST** `/api/content/queue/{publication_id}/approve`
  - Действие: `approved` + публикация на сайт (и VK — если настроено).

- **POST** `/api/content/queue/{publication_id}/pin`
  - Действие: переключить флаг `pinned` (закрепление/открепление).

- **POST** `/api/content/queue/{publication_id}/cancel`
  - Действие: «Отмена» — убрать из очереди (архивация).

### Публичные новости

- **GET** `/api/content/site/index`
  - Возвращает `items[]` (публикации): `publication_id`, `title`, `published_at_utc`, `url`, `pinned`.

- **GET** `/api/content/site/{publication_id}`
  - Возвращает Markdown‑текст публикации (для сайта).

## UI требования

### `/publapprov/`

Для каждого материала — отдельный блок, в котором:

- **Заголовок** (редактируемое поле).
- **Два независимых окна**:
  - **Сайт**: редактируемый текст (Markdown).
  - **ВКонтакте**: редактируемый текст.
- **Кнопки**:
  - **Сохранить**: вызывает `PUT /api/content/queue/{id}`.
  - **Опубликовать**:
    - если отмечено **«Отмена»** → `POST /cancel` и элемент исчезает из списка
    - если отмечено **«Одобрить»** → `POST /approve` и элемент исчезает из списка (появится в `/news/`)
    - если отмечено **«Закрепить»** → перед/после публикации вызвать `POST /pin` (переключатель)
    - если ничего не отмечено → просто сохранить (остается в очереди)

### `/news/`

- Список новостей в порядке: **закреплённые сверху**, далее новые.
- Каждая новость раскрывается полным текстом (можно MVP: показывать Markdown как plain text).

### Главная `/`

- В блоке «Новости» показывать несколько последних (например 3–6):
  - дата
  - заголовок
  - признак закрепления (📎)
  - ссылка на `/news/#<publication_id>`

## Формат «### Общий формат публикаций»

Контент приходит уже в Markdown. Фронт обязан:

- не ломать разметку
- сохранять текст как есть (без скрытых преобразований)
- редактирование — в текстовых полях/textarea

## Примечания

- Почтовый сценарий Chief отключён: согласование происходит только через `/publapprov/`.
- Если VK не настроен, `approve` может сохранить `last_publish_error` — UI на этом этапе может просто показывать «Опубликовано на сайт» (VK обработаем отдельно).
