# Настройки почты (SMTP + IMAP) — простыми словами

Эта инструкция нужна, чтобы приложение могло:

- **отправлять письма Chief** (на согласование) через **SMTP**
- **читать ответы Chief** («ДА / НЕТ / РЕДАКТИРОВАТЬ…») через **IMAP**

Всё это задаётся в `.env`.

## Важно про безопасность (коротко)

- **Пароль от почты в `.env` — это секрет.** Не коммитьте `.env` в репозиторий.
- Если включена **двухфакторная аутентификация (2FA)**, чаще всего нужен **пароль приложения**, а не “обычный” пароль.
- Лучше завести отдельный ящик для робота (например `robot@...`), чтобы не светить личный.

## Что такое SMTP и IMAP (очень просто)

- **SMTP** = “отправка писем” (как почтовый “выход”).
- **IMAP** = “чтение писем” (как почтовый “вход”).

Приложение будет:

- отправлять письмо Chief → SMTP
- потом периодически проверять входящие и искать новые ответы → IMAP

## Параметры SMTP (отправка писем)

### `SMTP_HOST`
**Адрес сервера отправки**.

- Для Яндекс.Почты обычно: `smtp.yandex.ru`
- Для корпоративного домена на Яндексе обычно тоже: `smtp.yandex.ru`

### `SMTP_PORT`
**Порт** (как “номер двери” на сервере).

Чаще всего:

- `587` — SMTP с **STARTTLS** (рекомендуется)
- `465` — SMTP через **SSL/TLS сразу**

В нашем проекте по умолчанию стоит `587`.

### `SMTP_USER`
**Логин** для SMTP. Обычно это **ваш полный email**, например:

- `robot@your-domain.ru`
- `robot@yandex.ru`

### `SMTP_PASSWORD`
**Пароль для SMTP**.

Варианты:

- если 2FA **не включена**: может подойти обычный пароль от почты
- если 2FA **включена**: нужен **пароль приложения** (самый частый случай)

#### Где взять пароль приложения (Яндекс)
1) Откройте страницу управления аккаунтом Яндекс (паспорт).
2) Найдите раздел **“Пароли приложений”** (или “App passwords”).
3) Создайте пароль для почты (иногда это называется “Почта / IMAP / SMTP”).
4) Скопируйте сгенерированный пароль и вставьте в `SMTP_PASSWORD`.

Если пункта “пароли приложений” нет — значит нужно сначала включить 2FA или изменить настройки безопасности.

### `SMTP_FROM`
**Адрес “от кого”** в письме.

- Обычно можно оставить пустым — тогда будет использован `SMTP_USER`.
- Если хотите писать “красивее” (например, `noreply@...`), убедитесь, что почтовый сервер разрешает такой From для вашего аккаунта (иначе письмо может не уйти или From будет заменён).

### `SMTP_TLS`
Включать ли шифрование **STARTTLS** при соединении.

- Для порта `587` обычно: `true`
- Для порта `465` обычно STARTTLS **не нужен** (но в нашем коде сейчас логика простая: если `SMTP_TLS=true`, он делает `starttls()`).

Практический совет:
- если используете `SMTP_PORT=587` → ставьте `SMTP_TLS=true`
- если используете `SMTP_PORT=465` → поставьте `SMTP_TLS=false` (или поменяйте порт на 587)

## Параметры IMAP (чтение ответов)

### `IMAP_HOST`
**Адрес сервера чтения почты**.

- Для Яндекс.Почты обычно: `imap.yandex.ru`

### `IMAP_PORT`
**Порт IMAP**.

- Обычно: `993` (IMAP через SSL/TLS)

### `IMAP_USER`
**Логин** для IMAP — обычно полный email, например `chief@yandex.ru`.

Важно: в нашем контуре **IMAP нужен для ящика Chief**, потому что именно оттуда мы читаем его решения.

### `IMAP_PASSWORD`
**Пароль для IMAP**.

Правила те же, что и для SMTP:

- если 2FA включена → чаще нужен **пароль приложения**
- если 2FA выключена → может подойти обычный пароль

### `IMAP_FOLDER`
**Папка**, которую мы читаем.

- Обычно: `INBOX`

Если у Chief ответы приходят в другую папку/правило (например, “Входящие/Согласования”), можно указать её имя. На первом этапе лучше оставить `INBOX`.

## Настройки на стороне Яндекс.Почты (что включить)

Чтобы IMAP работал, в почте часто нужно включить доступ по IMAP.

Для Яндекс.Почты обычно делается так:

1) Откройте Яндекс.Почту в браузере.
2) Зайдите в **Настройки** (шестерёнка).
3) Найдите раздел про **почтовые программы** / **IMAP**.
4) Включите **IMAP** (и сохраните).

Если в настройках безопасности включена защита, Яндекс может потребовать:
- подтвердить вход,
- создать пароль приложения,
- разрешить доступ почтовым программам.

## Готовый пример `.env` (Яндекс)

Ниже пример для случая, когда:

- мы **отправляем** письма с робо‑ящика `robot@...`
- **читаем** ответы из ящика Chief `v.devops@yandex.ru`

Поменяйте логины/пароли на свои.

```env
# Кому отправлять на согласование
CHIEF_EMAIL_TO=v.devops@yandex.ru

# SMTP (отправка писем Chief)
SMTP_HOST=smtp.yandex.ru
SMTP_PORT=587
SMTP_USER=robot@your-domain.ru
SMTP_PASSWORD=ВАШ_ПАРОЛЬ_ПРИЛОЖЕНИЯ_ИЛИ_ОБЫЧНЫЙ
SMTP_FROM=robot@your-domain.ru
SMTP_TLS=true

# IMAP (чтение ответов Chief)
IMAP_HOST=imap.yandex.ru
IMAP_PORT=993
IMAP_USER=v.devops@yandex.ru
IMAP_PASSWORD=ПАРОЛЬ_ПРИЛОЖЕНИЯ_CHIEF
IMAP_FOLDER=INBOX
```

## Частые ошибки и как понять что не так

- **SMTP: “authentication failed / login failed”**
  - неверный `SMTP_USER` или `SMTP_PASSWORD`
  - нужен пароль приложения (особенно если включена 2FA)
- **IMAP: “LOGIN failed”**
  - тот же кейс: пароль/логин неверный или нужен пароль приложения
  - IMAP не включён в настройках почты
- **Письмо уходит, но From “не тот”**
  - сервер переписывает From, если вы поставили `SMTP_FROM`, который не разрешён
- **`poll-chief` ничего не находит**
  - письма уже “прочитаны” (мы ищем `UNSEEN`)
  - Chief ответил не с того адреса (мы фильтруем по адресу Chief)
  - в теме/теле нет `(00035)` — система не может привязать решение к публикации

## Готовые host/ports для Gmail

### SMTP (отправка)
- **`SMTP_HOST`**: `smtp.gmail.com`
- **`SMTP_PORT`**:
  - `587` (STARTTLS, рекомендовано)
  - `465` (SSL/TLS сразу)

### IMAP (чтение)
- **`IMAP_HOST`**: `imap.gmail.com`
- **`IMAP_PORT`**: `993` (SSL/TLS)
- **`IMAP_FOLDER`**: обычно `INBOX`

### Важные нюансы Gmail
- Почти всегда нужен **пароль приложения**:
  - включите 2FA в Google‑аккаунте
  - создайте **App password** (для “Mail”)
  - используйте его в `SMTP_PASSWORD` и/или `IMAP_PASSWORD`
- В интерфейсе Gmail проверьте, что **IMAP включён**:
  - Gmail → Settings → “See all settings” → “Forwarding and POP/IMAP” → IMAP Access → Enable IMAP
