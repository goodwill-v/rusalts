# Таблица агентов: рекомендуемые модели и взаимодействие

> Принцип: “минимальная достаточность и экономия”. Дорогие модели включать только на сложных задачах (регуляторика/сведение источников/важные релизы).

| Агент/роль | Рекомендуемые модели (Cursor/RouterAI) | Рекомендации вам по взаимодействию |
|---|---|---|
| Тимлид | `GPT-5.2 High` (архитектура/ТЗ/ревью), `Sonnet 4.6` (обсуждение вариантов/трейд‑оффов), `GPT-5.2 Fast` (быстрые правки/сводки) | Давайте входные ограничения (форматы, папки, контракты) и просите “артефакты в репо” (файлы в `docs/`). Для спорных решений просите 2 варианта с trade‑offs. |
| 1) Бэк (Backend) | `GPT-5.2 Codex` (основная разработка), `GPT-5.2 Codex Fast` (рефактор/рутинные правки), `GPT-5.2 High` (legal/комплаенс и аккуратные ответы) | Всегда требуйте: (а) ссылки на статьи БЗ и источники, (б) логи usage/cost, (в) стабильный API‑контракт. На legal‑темы просите режим “цитирование + дата актуализации”. |
| 2) Фронт (Frontend) | `GPT-5.2 Codex Fast` (JS/CSS/верстка), `Sonnet 4.5` или `Sonnet 4.6` (UX/структура страниц), `GPT-5.2` (сложные интеграции/обобщение) | Просите макеты/контракты UI (какие поля приходят, какие состояния), и чек‑лист VK (CSP, init, resize). Фиксируйте “Definition of Done” (доступность, мобильный VK). |
| 3) Парс (Parser) | `GPT-5.2 Fast` (классификация/дедуп/рутина), `GPT-5.2` (суммаризация диффов), `GPT-5.2 High` (сложные изменения/регуляторика) | Просите “change package” как единственный результат дня + отчёт: что обновилось, куда в БЗ положено, какие источники недоступны. Требуйте 0 выдумок, только извлечённое. |
| 4) Конт (Content) | `Sonnet 4.5` или `Sonnet 4.6` (тексты и стиль), `GPT-5.2 Fast` (адаптация под площадку/варианты), `GPT-5.2 High` (важные релизы/чувствительные темы) | Давайте чёткий тон: длина, стиль, CTA, запреты. Просите 3 версии: сайт (подробнее), VK (коротко), внутренняя (технично). |

## Привязка к RouterAI

В `.env` стоит хранить выбранные model id, например:
- `ROUTERAI_CHEAP_MODEL=...`
- `ROUTERAI_CHAT_MODEL=...`
- `ROUTERAI_REASONING_MODEL=...`
- `ROUTERAI_EMBEDDINGS_MODEL=...`

Конкретные идентификаторы моделей добавим после сверки списка доступных в RouterAI.
