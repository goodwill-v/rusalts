# АЛТ‑эксперт (`/api/chat`): алгоритм и источники

## Цепочка обработки запроса

1. **Триггеры шаблонов** (`knowledge_base/triggers.json` + `templates/alt_expert_ru.json`) — если сработал триггер, ответ готов без LLM.
2. **Поиск в базе знаний** — `kb.search()` по `knowledge_base/articles/**/*.md` (YAML‑шапка + текст).
3. **Если есть хиты БЗ** — в ответ попадают выдержки; опционально **RouterAI** синтезирует связный текст **только по этим выдержкам** (policy **main → heavy**, см. `docs/MODEL_ROUTING.md`).
4. **Если хитов БЗ нет** — обход **официальных URL** из `parser/sources.json`: страницы скачиваются, ищется совпадение токенов запроса, фрагменты передаются в LLM с требованием ссылок (URL).
5. **Если вопрос в тематике проекта** (`is_alt_project_topic` в `app/model_routing.py`: MAX/миграция/VK/Telegram/боты/SBP/Госуслуги и т. п., либо регуляторика по триггерам) **и** включён веб (`WEB_SEARCH_ENABLED`, по умолчанию true) — **краткие выдержки из открытого веба** через DuckDuckGo: сначала Instant Answer API, при малом ответе — разбор HTML‑выдачи (`app/web_search.py`), затем LLM даёт **короткий** ответ с предупреждением перепроверить источники.
6. **Иначе** — нейтральное сообщение «обратитесь в поддержку» без веб‑поиска (вопрос вне тематики АЛТ).

## Policy моделей main → heavy

Реализовано в `app/model_routing.py` и используется в `app/routers/api.py`:

- по умолчанию **main** (`BACKEND_MODEL_MAIN`);
- **heavy** (`BACKEND_MODEL_HEAVY`) при юридическом/комплаенс‑триггере в тексте, при ровно одном слабом хите БЗ, либо при **пустом ответе** модели (один ретрай).

## Переменные окружения

| Переменная | Назначение |
|------------|------------|
| `BACKEND_MODEL_MAIN` | Основная модель чата |
| `BACKEND_MODEL_HEAVY` | Эскалация для сложных/юридических случаев и ретраев |
| `WEB_SEARCH_ENABLED` | `true`/`false` — веб‑fallback для тематики АЛТ (по умолчанию `true`) |

Системный промпт: `АЛЬТЕРНАТИВА_АЛТбот/ALT_sist.prompt.md`.
