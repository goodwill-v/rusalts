# Агент 3: Parser (ежедневный мониторинг и обновление БЗ)

## Цель

Ежедневно в **05:00 UTC** мониторить список официальных ресурсов, находить изменения, обновлять Базу знаний по секциям и формировать пакет изменений для Content‑агента.

## Входы

- список источников “официальные ресурсы” (после загрузки будет храниться в репозитории как таблица/JSON)
- текущее состояние БЗ: `knowledge_base/`
- прошлые снапшоты/диффы: `data/monitoring/` (runtime)

Нормализованный список источников:
- `parser/sources.json` (сгенерирован из `АЛЬТЕРНАТИВА_АЛТбот/Ресурсы_для_ежедневного_мониторинга_обновлений.xlsx`)
- схема: `parser/sources.schema.json`

## Выходы

- обновлённые статьи БЗ (атомарные `.md` + метаданные)
- `knowledge_base/changelog/*` (JSONL + MD)
- `data/changes/YYYY-MM-DD/change_package.json` (Parser → Content)

## Алгоритм (MVP)

1. Загрузить список источников → нормализовать в `sources.json`:
   - `id`, `title`, `url`, `category`, `expected_update_frequency`
2. Скачивание (с заголовками ETag/If‑Modified‑Since).
3. Очистка HTML до “основного текста” (readability/boilerplate removal).
4. Дифф с последней версией (по `id` источника).
5. Классификация изменений:
   - `legal.*` (приоритет)
   - `tech.*` (API/SDK/changelog)
   - `product.*`
   - `news.*`
6. Обновление БЗ:
   - добавить/обновить статьи по `section_path`
   - обязательно добавить источники и `updated_at_utc`
7. Сформировать `change_package.json` и передать Content‑агенту.

## Критичные правила

- не выдумывать факты: только извлекать из источников
- каждое изменение должно иметь `links[]` на официальные источники
- регуляторика — отдельная секция и отдельная хронология

## План инспекции после первого парсинга

- проверить, что изменения попали в “правильные” секции
- найти дубли и объединить статьи
- убедиться, что legal‑разделы имеют источники и дату вступления (если указана)

## Краткий бриф (для раздачи роли “Parser”)

Источники:
- `parser/sources.json` (нормализовано из `АЛЬТЕРНАТИВА_АЛТбот/Ресурсы_для_ежедневного_мониторинга_обновлений.xlsx`)
- `parser/sources.schema.json`

Ежедневный результат (05:00 UTC):
- обновление `knowledge_base/articles/*` по секциям (см. `docs/KNOWLEDGE_BASE.md`)
- журнал `knowledge_base/changelog/*`
- `data/changes/YYYY-MM-DD/change_package.json` для Content‑агента

Запреты:
- 0 выдумок: только извлечённое из источников
- каждое изменение → ссылки на официальный источник + `updated_at_utc`
