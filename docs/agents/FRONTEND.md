# Агент 2: Frontend (сайт, виджет VK, каналы коммуникаций)

## Цели

- развивать веб‑страницу консультационного сервиса (rusalts.ru) и виджет;
- обеспечить корректную работу в VK WebView/iframe;
- подготовить UI под новости/релизы и выдачу шаблонов.

## MVP

- UI “Чат + шаблоны” (уже есть в прототипе)
- блок “Новости/изменения” (появится после парсинга + контента)
- адаптация стилей под бренд АЛЬТЕРНАТИВА и требования VK

## Интеграции

- VK mini app: `VKWebAppInit`, resize, корректные CSP/CORS
- сайт: статика + обратный прокси на FastAPI (см. `deploy/`)

## Технические требования

- не утекать секретами: ключи VK и RouterAI — только на бэкенде
- работать в iframe: без `X-Frame-Options: DENY`, с `frame-ancestors`
- доступность: aria‑метки, роль `log` для чата, readable контраст

## План развития UI

- табы: “Консультация / Шаблоны / Новости / Вопросы‑ответы”
- источники: показывать “актуально на дату” и ссылки на официальные ресурсы
- кнопки быстрых действий (“Инструкция”, “Чек‑лист”, “Миграция бизнеса”)

## Краткий бриф (для раздачи роли “Frontend”)

Что уже есть:
- `app/templates/widget.html`, `app/templates/index.html`
- `app/static/js/app.js`, `app/static/css/app.css`
- VK инициализация: `VKWebAppInit` в `widget.html`
- CSP/CORS: `app/middleware.py`

MVP+ задачи:
- быстрые кнопки (“Инструкция”, “Чек‑лист”, “Миграция бизнеса”, “Законы/правила”)
- вкладка/панель “Новости” (после появления данных от Parser/Content)
- состояния UI: загрузка/ошибка/нет данных
- VK‑совместимость: resize после изменения высоты, проверка iframe/WebView

Ограничения:
- секреты не уходят на фронт (RouterAI/VK ключи — только сервер)
- не ломать iframe‑встраивание (без `X-Frame-Options: DENY`)
