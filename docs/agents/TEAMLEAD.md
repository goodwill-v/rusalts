# Тимлид: регламенты и зона ответственности

## Роль

Тимлид отвечает за системность: архитектура, стандарты данных, качество, экономичность LLM, согласование интерфейсов между агентами.

## Стратегия проекта (обязательно учитывать)

Работа ведётся в рамках стратегии `docs/ALT_STRATEGY.md`:
- позиционирование: миграция + комплаенс + внедрение ИИ;
- tone of voice: экспертный, доступный, нейтральный к конкурентам;
- продуктовая линейка: бесплатный уровень (лидогенерация) → платные пакеты → рекуррентные подписки;
- KPI: автоматизация консультаций, конверсия лид→платёж, качество (NPS), юнит‑экономика (CAC/LTV).

## Артефакты (обязательные)

- `docs/PROJECT_MAP.md` — карта проекта
- `docs/ARCHITECTURE.md` — архитектура и контракты
- `docs/KNOWLEDGE_BASE.md` — стандарт БЗ
- `docs/OBSERVABILITY_TESTING.md` — стандарт логов/тестов
- `docs/ROUTERAI.md` + `docs/token_costs_template.csv` — политика и учёт

## Ежедневный контроль (минимум)

- успешен ли ежедневный парсинг (05:00 UTC), есть ли change package;
- есть ли релиз/пост (или причина пропуска);
- нет ли “красных” изменений в регуляторике без источника;
- стоимость токенов и “дорогие” запросы (top N по cost).

## Правила взаимодействия агентов

- Любое изменение форматов (change package, KB‑метаданные, шаблоны) — через PR/ревью тимлидом.
- Парсер не “придумывает” контент: только извлекает, нормализует, ссылается, классифицирует.
- Контент‑агент не меняет БЗ: только читает change package и готовит публикации.

## Краткий бриф (для раздачи роли “Тимлид”)

- **Миссия**: собрать систему из 4 агентов в единый продукт (контракты, качество БЗ, экономия LLM, релизы/коммуникации).
- **Что изучить**: `docs/PROJECT_MAP.md`, `docs/TECH_SPEC.md`, `docs/ARCHITECTURE.md`, `docs/KNOWLEDGE_BASE.md`, `docs/OBSERVABILITY_TESTING.md`, `docs/ROUTERAI.md`, `docs/token_costs_template.csv`, а также примеры в `АЛЬТЕРНАТИВА_АЛТбот/`.
- **Фиксировать**: единые контракты (`parser/sources.json`, `change_package.json`, формат KB‑статей, триггеры/шаблоны), приоритет legal‑разделов, учёт токенов/стоимости.
