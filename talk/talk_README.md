## Что такое `/talk`

`/talk` — это **скрытая страница‑приложение** (её нет в меню сайта), которая открывается по адресу:

- `https://rusalts.ru/talk`

Она нужна, чтобы **общаться** с внешними/сторонними приложениями через простой “чат”, а также отправлять **вложения** (текстовые файлы, изображения, видео).

Важно: страница **не показывает** шапку/футер/логотипы — только чат, поле ввода и кнопку вложений.

## Как устроена безопасность

- Доступ к странице защищён **ключом `TALK_KEY`**.
- При первом входе браузер попросит ввести ключ.
- После успешного ввода ключ **сохраняется в браузере** (в `localStorage`) и окно ввода больше не появляется.
- Все запросы к внешним приложениям идут **не напрямую из браузера**, а через сервер АЛТ (это нужно для безопасности и чтобы работать с локальными адресами).
- Разрешены **только 3 адреса** (allowlist) из переменной `TALK_ALLOWED_URLS`.

## Что нужно настроить (администратору)

1) Откройте файл `talk/talk_example.env` и заполните:

- `TALK_KEY` — придумайте секретный ключ.
- `TALK_ALLOWED_URLS` — укажите **ровно 3** разрешённых адреса сторонних приложений (через запятую).

2) Перенесите эти переменные в основной `.env` проекта (рядом с остальными переменными) и перезапустите сервис.

## Как зайти на страницу (пользователю)

1) Откройте в браузере `https://rusalts.ru/talk`
2) Введите ключ `TALK_KEY`
3) Пишите сообщение в нижнем поле и нажимайте “Отправить”
4) Чтобы отправить файл — нажмите кнопку “+” и выберите файл

## Как подключить стороннее приложение (разработчику)

Стороннее приложение должно поднять HTTP endpoint (URL), который вы добавите в `TALK_ALLOWED_URLS`.

Страница `/talk` отправляет запросы на этот URL через сервер АЛТ:

- **Только текст**
  - запрос: `POST /api/talk/relay` (JSON)
  - сервер АЛТ форвардит в приложение: `POST <allowed_url>` с JSON `{"text": "..." }`

- **Текст + файл**
  - запрос: `POST /api/talk/relay-file` (multipart/form-data)
  - сервер АЛТ форвардит в приложение: `POST <allowed_url>` как multipart с полями:
    - `text` (строка)
    - `file` (файл)

Ожидаемый ответ приложения — JSON. Лучше всего вернуть:

```json
{ "reply": "Текст ответа, который появится в чате" }
```

## Как стороннему приложению отправить сообщение/файл в `/talk` (входящие)

Если нужно, чтобы приложение **само** прислало сообщение/файл пользователю на странице `/talk`, используйте endpoint АЛТ:

- `POST /api/talk/incoming` (multipart/form-data)

Заголовок авторизации:

- `X-Talk-App-Token: <TALK_APP_TOKEN>`  
  (если `TALK_APP_TOKEN` не задан — используйте `TALK_KEY`)

Поля формы:

- `text` — текст сообщения (можно пустой, если есть файл)
- `file` — файл (можно пустой, если есть текст)

Ответ:

- `{ "ok": true, "id": "..." }`

Пользователь на странице `/talk` увидит сообщение/файл и получит всплывающее оповещение.

### Как защитить стороннее приложение

Рекомендуемый вариант: **проверять секрет** на стороне приложения.

Самый простой способ:

- сделать в приложении дополнительный токен/ключ (например `X-App-Key`)
- и принимать запросы только при совпадении
