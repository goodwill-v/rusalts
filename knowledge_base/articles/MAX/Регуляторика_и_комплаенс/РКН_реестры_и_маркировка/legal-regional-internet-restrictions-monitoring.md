Статья описывает **организационную и техническую** сторону: как отслеживать ограничения доступа к ресурсам и что заложить в регламенты. Это не инструкция по обходу блокировок: создание зеркал без устранения основания блокировки может усугубить правовые риски.

## Типы ограничений (ориентир)

| Тип | Основание (примерно) | Примеры |
| --- | --------------------- | ------- |
| Судебные решения | Экстремизм, иные нарушения | Блок URL/домена |
| Требования РКН | Запрещённая информация, ПДн и др. | Записи в реестрах РКН |
| Региональные акты | Особенности субъектов РФ | Локальные требования к контенту/доступу |
| Инфраструктура провайдеров | Исполнение закона | Фильтрация, маршрутизация |

## Как проверять доступность и реестры

1. **Единый реестр запрещённой информации (ЕАИС)** — поиск по домену и основаниям: https://eais.rkn.gov.ru/
2. **Сервисы доступности** — проверки «из разных городов» сторонними мониторами (оценочно, не юридическое заключение).
3. **Open Data РКН** — выгрузки для автоматического мониторинга: https://rkn.gov.ru/opendata/

## Практика для запуска сервиса

1. Проверка домена и публичного контента на типовые запреты.
2. Локализация ПДн — см. статьи раздела `ПДн_152-ФЗ_и_хранение_в_РФ`.
3. Регламент реагирования: ответственный, чек-лист, коммуникации с пользователями.
4. Резервные каналы: отечественные мессенджеры (MAX, ВКонтакте), email, телефон — по политике продукта.

## Чек-лист региональной доступности

```

[ ] ПДн хранятся в РФ (если применимо)

```

## Шаблон: процедура реагирования на внесение в реестр

```
ПРОЦЕДУРА РЕАГИРОВАНИЯ НА ВНЕСЕНИЕ РЕСУРСА В РЕЕСТР

1. Обнаружение: eais.rkn.gov.ru, opendata, провайдер.
2. Анализ: основание, масштаб (URL/домен, география), перспектива обжалования.
3. Действия: обжалование/удаление контента/заявка на исключение — по стратегии юристов.
4. Коммуникации: уведомление пользователей, альтернативные каналы.
5. Документирование: карточка инцидента, хранение копий документов.
```

## FAQ

**Как быстро узнать о внесении в реестр?**  
Автоматический мониторинг Open Data + периодическая ручная проверка; уведомление хостера может задержаться.

**Можно ли обжаловать?**  
Зависит от основания; процедуры разные для судебных актов и административных требований — нужен юрист.

**Нужна ли маркировка «только для региона»?**  
Отдельной универсальной маркировки «регион» закон не задаёт; для детского контента действуют иные правила возрастной маркировки.

**Что если заблокирован домен?**  
Устранение основания, юридические шаги, **легальные** альтернативные каналы; не создавать зеркала нарушения.
